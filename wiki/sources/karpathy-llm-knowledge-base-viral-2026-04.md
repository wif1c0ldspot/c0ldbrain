## Summary

A thread analyzing what made Karpathy's LLM Obsidian knowledge base pattern go viral. Identifies 7 important characteristics of the self-improving wiki system.

## Key Points

1. **File-native storage** — Everything stored as markdown files, no database
2. **Verbatim capture** — Raw content stored, not extracted facts
3. **Agent-readable** — LLMs can read, write, and link files
4. **Compounding context** — Each session builds on previous
5. **Self-improving** — System gets better as it learns
6. **Zero manual editing** — Agent handles organization
7. **Context over retrieval** — Works inside context, not just searching

## Viral Impact

This pattern inspired many implementations including:
- C0ldbrain wiki (this vault)
- OpenClaw memory system
- Various personal knowledge bases
