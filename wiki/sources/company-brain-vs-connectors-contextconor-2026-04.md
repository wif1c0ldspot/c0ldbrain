## Summary

Conor Brennan-Burke (@contextconor) argues the industry framed agent context wrong — we built retrieval infrastructure (RAG, MCP, vector DBs) when we should be building synthesized understanding. A "company brain" that resolves conflicts, tracks identity across sources, monitors information decay, and surfaces as files any agent can read. Retrieval gives fragments; synthesis gives a worldview. Hyperspell is building this.

## Key Points

- **Access ≠ Understanding**: Agents can search all your tools but still don't understand your company. Retrieval is a scavenger hunt starting from zero every time.
- **Retrieval vs Synthesis**: Retrieval finds answers when asked. Synthesis maintains a continuously updated representation so answers already exist before anyone asks.
- **5 unsolved problems**: (1) Source conflict resolution, (2) Entity identity resolution across tools, (3) Information decay tracking, (4) Source authority hierarchy, (5) Cross-source synthesis (combining sources into insight none contain individually)
- **Filesystem as delivery**: Context graph surfaces as files on disk — the one interface every agent already supports. Decoupled from any vendor/framework.
- **Compounding moat**: Understanding accumulates day over day. Can't be bought or fast-forwarded. Proprietary to each company.
- **Missing benchmark**: No benchmark tests organizational context synthesis. They're building one — early results "humbling."
- **2025 = Access. 2026 = Understanding.**
