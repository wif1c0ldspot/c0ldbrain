## Summary

Discovery of the `/model opus` command in Claude Code that enables Opus-class model usage for planning complex tasks.

## The Feature

### Command
```
/model opus
```

### Purpose
Switch to Opus model (most capable) for:
- Complex planning
- Architecture decisions
- Strategy formulation
- Difficult problems

### Benefit
- Better reasoning for complex tasks
- More thorough planning
- Reduced iteration
- Higher quality output

## Usage Pattern

```
/model opus
[Describe complex task/architecture]

/model [switch back to faster model]
[Implement based on plan]
```

## When to Use

### Good For
- System architecture
- Multi-file refactors
- Complex debugging
- Architecture decisions
- Strategy planning

### Not Needed For
- Simple tasks
- Quick fixes
- Routine edits
- One-liners

## Cost Consideration

Opus is more expensive but worth it for:
- High-value decisions
- Complex architecture
- Long-term projects
- Difficult problems
