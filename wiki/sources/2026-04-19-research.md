## Pattern of the Day: **Demand Paging for Agent Memory**

The most intellectually significant finding today is the "Missing Memory Hierarchy" paper (ArXiv 2603.09023). It reframes agent memory management through the lens of **operating system virtual memory**:

- **Context window** = physical RAM (limited, fast)
- **External memory store** = disk (vast, slower)
- **Demand paging** = load context pages only when needed
- **Working set** = the subset of context actively being used
- **Eviction policy** = what to drop when context fills up (LRU, LFU, relevance-based)
- **Page fault** = retrieval call when needed context has been evicted

This is a **Camp 1.5 hybrid** approach — it uses Camp 1 storage techniques but thinks in Camp 2 terms (what context should the agent work inside?). The Pichay implementation is a transparent proxy between the agent and the LLM, managing context windows programmatically.

**Why this matters**: Every production agent system implicitly does demand paging. This paper makes it explicit, rigorous, and optimizable. Expect this framing to influence Mem0, Cognee, and Cloudflare's implementations.
