## Pattern of the Day: Claim-Based Memory with Authority Proofs

The shift from **summary-based memory** to **claim-based memory** is accelerating. Two tools discovered today exemplify this:

### chum-mem's PCKC Model
- Memory units are typed **claims**: `fact`, `decision`, `task`, `constraint`, `bug`, `fix`, `implementation_detail`, `open_question`
- Each claim carries **proof**: `authority_class` (tool_verified, user_confirmed, repository_derived, session_derived, model_inferred), `verification_status`, `source_ref`
- **Belief gate** rejects model-generated prose — only tool-verified or user-confirmed claims become durable memory
- **Supersession engine** tracks temporal validity — old claims marked when replaced

### cavemem's Compression Approach
- "Caveman grammar" compresses prose ~75% while preserving code/paths byte-for-byte
- Round-trip-guaranteed expansion for humans
- FTS5 keyword + local vector index hybrid search
- MCP tools: `search`, `timeline`, `get_observations`

### Why This Matters
These represent two poles of the "truth layer" problem:
1. **cavemem**: Compress everything, retrieve efficiently (Camp 1.5 hybrid)
2. **chum-mem**: Filter aggressively at write time, only store verified claims (Camp 1 + security)

The convergence point: **memory quality > memory quantity**. Both tools prioritize what's stored over how much is stored.

## Mem0 Architecture Evolution

### v2.0.0 Python SDK (April 16, 2026)
**Major architectural changes:**
1. **Single-pass ADD-only extraction** — roughly half the latency of previous multi-pass
2. **Multi-signal hybrid retrieval** — semantic + BM25 keyword + entity matching fused into one score
3. **Built-in entity linking** — replaces graph memory, no external store needed
4. **Cleaner SDK surface** — constructor and method signature cleanup

### OpenClaw Plugin v1.0.7 (April 20, 2026)
- Chat-based Platform setup flow — configure conversationally instead of editing config
- Dropped deprecated `orgId`, `projectId`, `enableGraph` parameters

**Strategic significance**: Mem0 is consolidating — replacing external graph stores with built-in entity linking, simplifying the API surface. This is the "Camp 1 wins production" thesis playing out.
