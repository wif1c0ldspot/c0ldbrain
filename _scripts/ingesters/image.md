# Image Ingester

## Process

- Use vision to describe: components, connections, data flows, text content
- For architecture diagrams: extract components AND relationships
- For screenshots: what it shows, relevant text, why it was captured
- Always create a sidecar .md with the description

## Rules

- Describe the image in text — wiki should be useful without the visual
- Include image reference: `![[filename]]`
- If from known source (paper, blog), link to that wiki page
