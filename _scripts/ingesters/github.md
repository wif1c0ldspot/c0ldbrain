# GitHub Ingester

## Extraction

- **Repo:** purpose, architecture, key APIs, dependencies, patterns (never dump raw code)
- **Issue/PR:** problem, root cause, solution, affected components
- Focus on WHY (design decisions) not just WHAT

## Wiki Mapping

- Repo docs → wiki/sources/<repo-slug>.md + concepts for architectures
- Issues → wiki/sources/<issue-slug>.md + concepts for problem patterns
